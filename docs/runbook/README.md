# Runbook (Operational)

This folder contains operational procedures for:
- Investigating data quality failures
- Handling drift alerts (data drift/model drift)
- Rolling back model versions
- Triggering safe retraining
- Root cause analysis (RCA)

## Minimum runbook structure
- **Symptom**
- **Checks**
- **Likely causes**
- **Mitigation**
- **Rollback**
- **Post-incident actions / RCA**
